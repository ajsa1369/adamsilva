Export `TIER_UNIT_COSTS` as a const:
```typescript
export const TIER_UNIT_COSTS = {
  1: { setup: 750,  monthly: 150 },
  2: { setup: 1500, monthly: 250 },
  3: { setup: 4000, monthly: 500 },
} as const
```

Export `calculatePricing(packageSlug: string, integrations: IntegrationSelection[]): PricingResult`.

Algorithm (implement exactly — this is LOCKED logic):
1. Look up the package from PACKAGES by slug. Throw `Error(`Unknown package slug: ${packageSlug}`)` if not found.
2. Initialize three slot buckets from the package: `t1Remaining`, `t2Remaining`, `t3Remaining` (copy from package.tier1Slots etc.)
3. Initialize `includedIntegrations: IntegrationSelection[] = []` and `overageIntegrations: IntegrationSelection[] = []`
4. Initialize `overageSetup = 0`, `overageMonthly = 0`
5. For each integration in the input array:
   - Determine its tier (1, 2, or 3)
   - If the corresponding `tN Remaining` bucket > 0:
     - Decrement `tNRemaining` by 1
     - Push to `includedIntegrations`
   - Else (bucket is 0 or slot count is 99 — but 99 is unlimited so always included):
     - For Core package (all slots are 99), always push to `includedIntegrations` — no overages
     - For non-Core: push to `overageIntegrations`, add `TIER_UNIT_COSTS[tier].setup` to `overageSetup`, add `TIER_UNIT_COSTS[tier].monthly` to `overageMonthly`
6. Build and return `PricingResult`:
   ```
   {
     packageSlug,
     baseSetup: pkg.baseSetup,
     baseMonthly: pkg.baseMonthly,
     overageSetup,
     overageMonthly,
     setupTotal: pkg.baseSetup + overageSetup,
     monthlyTotal: pkg.baseMonthly + overageMonthly,
     includedIntegrations,
     overageIntegrations,
   }
   ```

Core (tier1Slots=99, tier2Slots=99, tier3Slots=99) never generates overages — any integration count up to 99 is included. The check for unlimited slots: `if (slots === 99)` always treat as included.

Implement the LOCKED recommendation logic in this exact priority order:

**Step 1 — Legacy platform check (FIRST, before any other logic):**
```
if LEGACY_PLATFORMS includes input.platform.toLowerCase().trim()
  → return { recommendedSlug: 'shopify-growth' if t1Count >= 3 OR t2Count >= 1 else 'shopify-starter', isLegacyPath: true, reasoning: 'Legacy platform detected — routing to legacy package path' }
```
(Use shopify-growth if integrations warrant it, shopify-starter otherwise)

**Step 2 — Enterprise ERP check:**
```
if any integration name (lowercased) matches ENTERPRISE_TOOLS
  → return { recommendedSlug: 'core', isLegacyPath: false, reasoning: 'Enterprise ERP detected (${toolName}) — Core tier required' }
```

**Step 3 — 10+ integrations check:**
```
if integrations.length >= 10
  → return { recommendedSlug: 'core', isLegacyPath: false, reasoning: '${integrations.length} integrations selected — Core tier required (10+ threshold)' }
```

**Step 4 — Zero integrations check:**
```
if integrations.length === 0
  → return { recommendedSlug: 'bronze', isLegacyPath: false, reasoning: 'No integrations selected — Bronze is the starting tier' }
```

**Step 5 — Goal/lead/location overrides (compute minimum tier, then apply):**
Start with `minSlug = 'bronze'`. Apply each override rule in order (each rule can only increase the minimum):
- Any T3 integration present → minSlug = minimum('gold', current min) → 'gold'
- goals includes 'ucp' OR 'acp' → minSlug = minimum('gold', current min) → 'gold'
- input.monthlyLeads >= 500 → minSlug = minimum('gold', current min) → 'gold'
- input.monthlyLeads >= 200 (and < 500) → minSlug = minimum('silver', current min) → 'silver'
- input.locationCount >= 6 → minSlug = minimum('silver', current min) → 'silver'
- Any T2 integration present → minSlug = minimum('silver', current min) → 'silver'

For tier ordering use: bronze < silver < gold < core. The function `upgradeTier(current, candidate)` compares and returns the higher.

**Step 6 — Slot-fit check (apply AFTER overrides):**
Starting from minSlug, check if the package can accommodate all integrations within slots (no overage) by calling `calculatePricing(slug, integrations)`.
- If `result.overageIntegrations.length === 0` → the minSlug fits, return it
- Otherwise try the next tier up (bronze→silver→gold→core) until a fitting package is found
- If even Core fits (it always does), return 'core'

Return:
```typescript
{
  recommendedSlug: finalSlug,
  isLegacyPath: false,
  reasoning: buildReasoning(rules applied, final recommendation)
}
```

The `reasoning` string should be a concise, human-readable summary of WHY this tier was chosen (e.g. "UCP goal requires minimum Gold; slot-fit check confirms Gold accommodates all integrations").

Strictly typed — no `any`. All comparisons use lowercase-normalized strings.
