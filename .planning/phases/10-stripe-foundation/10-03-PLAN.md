<objective>
Create the Stripe webhook endpoint at `/api/stripe/webhook` with signature verification and idempotency, plus the `stripe_events` Supabase migration for duplicate event detection.

Purpose: Stripe webhooks are the only way to reliably track payment outcomes (succeeded, failed, refunded). Without this endpoint, the system cannot confirm that a payment was actually received. The idempotency table prevents double-processing when Stripe retries webhook delivery (at-least-once guarantee).

Output: A working webhook route that accepts valid Stripe events, rejects invalid signatures, and deduplicates retries — plus the migration file to create the idempotency table.
</objective>
