- Table: integrations_catalog
- Columns:
  - id UUID PRIMARY KEY DEFAULT gen_random_uuid()
  - name TEXT NOT NULL — tool name (e.g. "HubSpot", "Stripe")
  - category TEXT NOT NULL — one of: crm, booking, email, payment, helpdesk, accounting, platform, erp
  - tier INTEGER NOT NULL CHECK (tier IN (1, 2, 3))
  - setup_cost NUMERIC(10,2) NOT NULL — Tier 1=$750, Tier 2=$1500, Tier 3=$4000
  - monthly_cost NUMERIC(10,2) NOT NULL — Tier 1=$150, Tier 2=$250, Tier 3=$500
  - api_available BOOLEAN NOT NULL DEFAULT true
  - connector_type TEXT — e.g. 'pre-built', 'custom-rest', 'legacy'
  - notes TEXT
  - created_at TIMESTAMPTZ DEFAULT NOW() NOT NULL
- Unique constraint: UNIQUE (name) — tool names must be unique
- Indexes: idx_integrations_catalog_tier (tier), idx_integrations_catalog_category (category), idx_integrations_catalog_name (name)
- RLS: Enable + service_role_all policy matching existing pattern

Write supabase/migrations/005_packages.sql:

- Columns:
  - id UUID PRIMARY KEY DEFAULT gen_random_uuid()

  - base_setup NUMERIC(10,2) NOT NULL
  - base_monthly NUMERIC(10,2) NOT NULL
  - tier1_slots INTEGER NOT NULL DEFAULT 0
  - tier2_slots INTEGER NOT NULL DEFAULT 0
  - tier3_slots INTEGER NOT NULL DEFAULT 0
  - features_json JSONB NOT NULL DEFAULT '[]'::jsonb
  - is_legacy_only BOOLEAN NOT NULL DEFAULT false
  - created_at TIMESTAMPTZ DEFAULT NOW() NOT NULL
- Indexes: idx_packages_slug (slug), idx_packages_is_legacy_only (is_legacy_only)
- RLS: Enable + service_role_all policy

Both files must use CREATE TABLE IF NOT EXISTS for idempotency. Do NOT include seed data in these files — seeds are in separate migration files (Plan 03).

- Columns:
  - id UUID PRIMARY KEY DEFAULT gen_random_uuid()
  - created_at TIMESTAMPTZ DEFAULT NOW() NOT NULL
  - prospect_name TEXT NOT NULL
  - prospect_email TEXT NOT NULL
  - industry TEXT
  - locations INTEGER DEFAULT 1
  - monthly_leads INTEGER

  - platform_tier TEXT — 'legacy' or 'headless'

  - monthly_total NUMERIC(10,2)
