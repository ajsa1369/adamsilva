**app/api/authority-map/generate/route.ts**

Server-side route (no 'use client'). Use `export const runtime = 'nodejs'`.

Imports:
- `import { z } from 'zod'` — for request body validation
- `import { generateAuthorityMap } from '@/lib/authority-map/researcher'`
- `import type { ClientConfig, AuthorityMapTopic } from '@/lib/authority-map/types'`

Zod schema for request body:
```typescript
const GenerateSchema = z.object({
  clientId: z.string().min(1),
  industry: z.string().min(1),
  serviceArea: z.string().min(1),
  approvalEmail: z.string().email(),
  existingContentUrls: z.array(z.string().url()).optional(),
})
```

POST handler flow:
1. Parse and validate request body with GenerateSchema. On error: return 400 `{ error: 'Invalid request', details: zErr.errors }`.
2. Build ClientConfig from validated body.
3. Call `const result = await generateAuthorityMap(config)` — this returns AuthorityMapResult.
4. Upsert to Supabase authority_maps via REST fetch (see Supabase upsert pattern in interfaces above). Body: `{ client_id: result.clientId, month: result.month, topics_json: result.topics }`.
5. Extract `row.id` (UUID) from `[row] = await res.json()`. If res.ok is false, throw with status text.
6. Send approval email via Resend direct API (see Resend pattern in interfaces).

Approval email HTML (branded #1B2E4B + #4D8EC0, matching Phase 4 send-proposal-email pattern):
- Subject: `Your ${monthLabel} Content Calendar is Ready — Review Required`
  - monthLabel: derive from result.month ('YYYY-MM') using split('-') → `${months[month-1]} ${year}` where months = ['January','February',...,'December']
- Body sections:
  - Header banner: background #1B2E4B, white text "Adam Silva Consulting — Content Calendar"
  - Intro paragraph: "Your [Month] topical authority map is ready for review. We've identified [N] high-opportunity content topics based on real-time competitor analysis and AI citation gap research."
  - Topics table: display top 5 topics (result.topics.slice(0,5)) with columns: Rank | Topic | Authority Gap Score | Est. Citation Lift
  - [Approve Content Calendar] button: background #4D8EC0, white text, links to `${process.env.NEXT_PUBLIC_SITE_URL}/api/authority-map/approve?id=${row.id}`
  - Modification note: "To modify topics, reply to this email with your changes. Content production begins once approved."
  - Footer: "Adam Silva Consulting | adamsilvaconsulting.com"

7. Return `{ success: true, mapId: row.id, topicCount: result.topics.length }` with status 200.

Wrap entire handler in try/catch — on error return 500 `{ error: err.message }`.

**app/api/authority-map/cron/route.ts**

Server-side route. Use `export const runtime = 'nodejs'`.

Imports:
- `import { generateAuthorityMap } from '@/lib/authority-map/researcher'`
- `import type { ClientConfig } from '@/lib/authority-map/types'`

GET handler flow:
1. Verify CRON_SECRET (same pattern as Phase 4 followup cron):
   ```typescript
   const secret = req.headers.get('authorization')?.replace('Bearer ', '')
   if (secret !== process.env.CRON_SECRET) {
     return Response.json({ error: 'Unauthorized' }, { status: 401 })
   }
   ```
2. Parse AUTHORITY_MAP_CLIENTS env var:
   ```typescript
   const raw = process.env.AUTHORITY_MAP_CLIENTS
   if (!raw) return Response.json({ processed: 0, errors: ['AUTHORITY_MAP_CLIENTS not set'] })
   const clients: ClientConfig[] = JSON.parse(raw)
   ```
3. For each client, call the full generation + persist + email flow. To avoid duplicating generate/route.ts logic, call the Supabase upsert and Resend email inline OR extract a shared helper. The simplest v1 approach: call the same shared library directly.

   Use a sequential loop (per deferred: no Promise.allSettled in v1 — avoids rate limits):
   ```typescript
   const errors: string[] = []
   let processed = 0
   for (const client of clients) {
     try {
       // 1. Run researcher
       const result = await generateAuthorityMap(client)
       // 2. Upsert to Supabase
       const supaRes = await fetch(
         `${process.env.NEXT_PUBLIC_SUPABASE_URL}/rest/v1/authority_maps`,
         {
           method: 'POST',
           headers: {
             'apikey': process.env.SUPABASE_SERVICE_ROLE_KEY!,
             'Authorization': `Bearer ${process.env.SUPABASE_SERVICE_ROLE_KEY!}`,
             'Content-Type': 'application/json',
             'Prefer': 'return=representation,resolution=merge-duplicates',
           },
           body: JSON.stringify({
             client_id: result.clientId,
             month: result.month,
             topics_json: result.topics,
           }),
         }
       )
       const [row] = await supaRes.json()
       // 3. Send approval email via Resend
       await fetch('https://api.resend.com/emails', {
         method: 'POST',
         headers: {
           'Authorization': `Bearer ${process.env.RESEND_API_KEY}`,
           'Content-Type': 'application/json',
         },
         body: JSON.stringify({
           from: 'Adam Silva Consulting <info@adamsilvaconsulting.com>',
           to: [client.approvalEmail],
           subject: `Your ${result.month} Content Calendar is Ready — Review Required`,
           html: `<p>Authority map generated. <a href="${process.env.NEXT_PUBLIC_SITE_URL}/api/authority-map/approve?id=${row.id}">Approve here</a></p>`,
         }),
       })
       processed++
     } catch (err) {
       errors.push(`${client.clientId}: ${err instanceof Error ? err.message : String(err)}`)
     }
   }
   return Response.json({ processed, errors })
   ```

   Note: The cron email HTML is minimal (simple link) — the full branded template is in the generate route. This keeps the cron route lean.
