  <action>
Create the POST handler for the chatbot streaming API. This is a Node.js runtime route (not edge) because it writes to Supabase for session persistence.

Full implementation requirements:

1. **Parse request body**: `{ messages: ModelMessage[], sessionId?: string, channel?: ChatChannel }`

2. **Extract last user message**: Find the last message with role='user' from messages array. Used for RAG retrieval.

3. **RAG retrieval**: Call `retrieveContext(clientId, lastUserMessage)` — non-fatal (returns '' if Supabase unavailable).

4. **Build system prompt**: Compose the chatbot system prompt. Include:
   - Role: "You are a helpful AI assistant for [clientId]. You help website visitors with questions, job cost estimates, appointments, and lead capture."
   - RAG context block (injected from retrieveContext result — insert at end of system prompt if non-empty)
   - Tool guidance: "When a visitor asks about pricing or costs, use the calculateJobCost tool. For booking, use bookAppointment. To capture lead info, use createCRMLead with crmType from client configuration. If visitor is frustrated or needs human help, use escalateToHuman. For order tracking, use lookupOrderStatus."

5. **streamText call**:
```typescript
const result = await streamText({
  model: getChatModel(),
  system: systemPrompt,
  messages,
  tools: chatbotTools,
  stopWhen: stepCountIs(10),
  onFinish: async ({ text, toolCalls }) => {
    // Persist session after streaming completes
    await persistSession(clientId, sessionId, channel, messages, text, toolCalls)
  },
})
return result.toUIMessageStreamResponse()
```

6. **persistSession helper** (defined in same file, async, non-fatal):
- Create a Supabase client with service role key
- Map messages to ChatMessage[] (extract text content from parts, add timestamp)
- Append the assistant's final response as a ChatMessage
- If toolCalls includes 'escalateToHuman' → outcome = 'escalated'
- If toolCalls includes 'bookAppointment' with success → outcome = 'booked', appointmentBooked = true
- If toolCalls includes 'createCRMLead' with success → outcome = 'qualified', crmContactId = contactId
- Upsert to chatbot_sessions: if sessionId provided, update existing row; else insert new row
- Wrap entire operation in try/catch — never throw; log errors to console.error

7. **Session ID flow**: The client generates a UUID sessionId and passes it on every request for the same conversation. The route upserts by sessionId to accumulate message history.

Use `runtime = 'nodejs'` (NOT 'edge') — required for Supabase createClient and session upsert.
Use `maxDuration = 30` — chatbot responses are shorter than intake agent.

DO NOT use `export const runtime = 'edge'` — that pattern is from intake/chat/route.ts which doesn't write to Supabase.
