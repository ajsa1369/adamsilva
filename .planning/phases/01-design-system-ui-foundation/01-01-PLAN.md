**Button.tsx:**

`'use client'` directive (uses onClick handlers).

```typescript
export interface ButtonProps {
  variant?: 'primary' | 'secondary' | 'ghost' | 'danger'
  size?: 'sm' | 'md' | 'lg'
  disabled?: boolean
  loading?: boolean
  type?: 'button' | 'submit' | 'reset'
  onClick?: () => void
  children: React.ReactNode
  className?: string
  href?: string  // renders as <a> when provided
  'aria-label'?: string
}
```

Implementation rules:
- Map variant to existing globals.css utility classes:
  - `primary` → `.btn-primary` class
  - `secondary` → `.btn-secondary` class
  - `ghost` → `.btn-ghost` class
  - `danger` → inline Tailwind classes: `bg-red-600 hover:bg-red-700 text-white font-semibold rounded-lg transition-all duration-200`
- Map size to padding/font-size Tailwind classes:
  - `sm` → `px-3 py-1.5 text-sm`
  - `md` → (default size already in btn-* classes, no override)
  - `lg` → `px-6 py-3.5 text-base`
- When `href` is provided, render as `<a>` element with the same styling
- When `loading` is true, render a small spinner SVG inline before children and set `disabled` behavior
- `disabled` state: `opacity-50 cursor-not-allowed pointer-events-none`
- Do NOT redefine the button CSS — apply the existing classes via `className` prop merge
- Mobile-first: default sizes work at 375px (no overflow)

**Card.tsx:**

Server component (no state/effects needed).

```typescript
export interface CardProps {
  variant?: 'default' | 'glass'
  hover?: boolean
  padding?: 'none' | 'sm' | 'md' | 'lg'
  children: React.ReactNode
  className?: string
  as?: 'div' | 'section' | 'article'
}
```

Implementation rules:
- `default` variant → `.card` class (already in globals.css)
- `glass` variant → `.card-glass` class (already in globals.css)
- `hover` true → hover styles already in `.card:hover` and `.card-glass:hover` — no extra classes needed
- padding values: `none` = no padding, `sm` = `p-4`, `md` = `p-6` (default), `lg` = `p-8`
- `as` prop defaults to `'div'`
- No hardcoded colors — all colors come from CSS variables via the existing `.card` / `.card-glass` classes
- Works correctly at 375px: ensure no fixed widths that would overflow

TypeScript: Strict mode, explicit return type `React.JSX.Element`, no `any`.
