Create lib/press-release/schema-builder.ts. Produces a 5-node @graph for press releases.

```typescript
import type { PressReleaseSchemaInput } from '@/lib/press-release/types'
import { SITE_URL, ORG_ID, organizationSchema } from '@/lib/schemas/organization'

export function buildPressReleaseSchema(input: PressReleaseSchemaInput): object[] {
  // Returns @graph array with exactly 5 nodes:
  // 1. NewsArticle — @id: input.pressReleaseUrl + '#article'
  // 2. Organization — @id: ORG_ID (import from lib/schemas/organization)
  // 3. ImageObject[] — one per input.images (spread into graph, @id: contentUrl + '#img')
  // 4. VideoObject — @id: input.pressReleaseUrl + '#video' (only if input.video present)
  // 5. BreadcrumbList — @id: input.pressReleaseUrl + '#breadcrumb'
  ...
}
```

**Node specifications:**

**NewsArticle node:**
```json
{
  "@type": "NewsArticle",
  "@id": "{pressReleaseUrl}#article",
  "headline": "{headline}",
  "name": "{headline}",
  "url": "{pressReleaseUrl}",
  "description": "{first 200 chars of body}",
  "datePublished": "{publishedAt}",
  "dateModified": "{publishedAt}",
  "author": { "@type": "Person", "name": "{authorName}", "@id": "{SITE_URL}/about#person" },
  "publisher": { "@id": "{ORG_ID}" },
  "isAccessibleForFree": true,
  "image": [{ "@id": "{images[0].contentUrl}#img" }],
  "video": { "@id": "{pressReleaseUrl}#video" },  // only if video present
  "archivedAt": "{wireResultUrl}",                // only if wireResultUrl present
  "keywords": "{topic} press release, agentic commerce, Adam Silva Consulting",
  "about": { "@type": "Thing", "name": "{topic}" },
  "speakable": {
    "@type": "SpeakableSpecification",
    "cssSelector": ["h1.pr-headline", ".pr-lead", ".pr-body"]
  }
}
```
`speakable` tells Google Assistant, voice AI, and LLMs which sections to read. CSS selectors reference the press release page layout (press release pages use `class="pr-headline"`, `pr-lead`, `pr-body`).

**Organization node:** Use `organizationSchema` from lib/schemas/organization.ts directly (already has all fields). Do NOT re-declare Organization fields inline.

**ImageObject nodes:** One per input.images entry:
```json
{
  "@type": "ImageObject",
  "@id": "{contentUrl}#img",
  "url": "{contentUrl}",
  "contentUrl": "{contentUrl}",
  "name": "Press Release: {headline} — Image {i+1}",
  "description": "Press release image for: {headline}. Published by Adam Silva Consulting.",
  "caption": "{headline}",
  "encodingFormat": "image/png",
  "width": 1200,
  "height": 630,
  "embeddedTextCaption": "{headline}"
}
```
`description` and `caption` allow LLMs without vision access to understand image content. XMP metadata (embedded by generateImages) mirrors these values.

**VideoObject node** (only if `input.video` is defined):
```json
{
  "@type": "VideoObject",
  "@id": "{pressReleaseUrl}#video",
  "name": "{headline} — Press Release Video",
  "description": "60-second press release summary video for: {headline}",
  "contentUrl": "{video.videoPath}",
  "thumbnailUrl": "{video.stillImageUrl}",
  "duration": "PT60S",
  "uploadDate": "{publishedAt}",
  "hasPart": { "@id": "{video.captionTrack.captionUrl}#caption" },
  "transcript": "{video.transcript}"
}
```
Duration is always "PT60S" (per PR-03 spec) — do NOT use `video.duration` from the pipeline (which may be shorter for stubs).

**BreadcrumbList node:**
```json
{
  "@type": "BreadcrumbList",
  "@id": "{pressReleaseUrl}#breadcrumb",
  "itemListElement": [
    { "@type": "ListItem", "position": 1, "name": "Home", "item": "{SITE_URL}" },
    { "@type": "ListItem", "position": 2, "name": "Press Releases", "item": "{SITE_URL}/press-releases" },
    { "@type": "ListItem", "position": 3, "name": "{headline}", "item": "{pressReleaseUrl}" }
  ]
}
```

Build the graph array: always include NewsArticle + Organization + BreadcrumbList. Spread ImageObject nodes (1 per image). Only add VideoObject if `input.video` is defined. Return as `object[]`.

Create lib/press-release/media-pipeline.ts. This is a thin orchestrator — it adapts press-release inputs to the Phase 6 pipeline inputs and sets press-release-specific output paths.

**Key differences from insights pipeline:**
- Output video directory: `public/videos/press-releases/` (not `public/videos/insights/`)
- Output image directory: `public/images/press-releases/` (not `public/images/insights/`)
- Video composition duration target: 60 seconds (PR-03) — the Remotion composition handles this
- Topic must be synthesized from the press release headline (press releases don't come from AuthorityMapTopic)
- Video description is a 60-word summary of the press release headline + lead

```typescript
import { generateImages } from '@/lib/insights/image-pipeline'
import { generateVideo } from '@/lib/insights/video-pipeline'
import { SITE_URL } from '@/lib/schemas/organization'
import type { ImagePipelineResult, VideoPipelineResult } from '@/lib/insights/types'
import type { PressReleaseDraft, PressReleaseMediaOptions } from '@/lib/press-release/types'

export interface PressReleaseMediaResult {
  images: ImagePipelineResult[]
  video?: VideoPipelineResult  // undefined if options.includeVideo === false
}

export async function generatePressReleaseMedia(
  draft: PressReleaseDraft,
  slug: string,
  authorName: string,
  options: PressReleaseMediaOptions,
): Promise<PressReleaseMediaResult>
```

**Implementation:**

1. Build a synthetic AuthorityMapTopic from the draft to satisfy the image/video pipeline input types:
```typescript
const syntheticTopic = {
  rank: 1,
  title: draft.headline,
  targetQueries: [draft.headline, draft.lead.slice(0, 80)],
  authorityGapScore: 0,
  recommendedSchemaTypes: ['NewsArticle'],
  faqClusters: [],
  estimatedCitationLift: '0',
}
```

2. If `options.includeVideo === true`:
   - Call `generateVideo({ topic: syntheticTopic, slug, postTitle: draft.headline, excerpt: draft.lead.slice(0, 240), authorName, outputDir: 'public/videos/press-releases' })`
   - Remotion still frame goes to `public/images/press-releases/[slug]-title.png`

3. Call `generateImages` with `imageCount: options.imageCount`, `imagesBaseUrl: ${SITE_URL}/images/press-releases`:
   - Override output directory: images go to `public/images/press-releases/` (not insights/)
   - Pass `video.stillImagePath` as stillFramePath if video was generated

4. Return `{ images, video }` — video is undefined if `includeVideo === false`

**Note on output directory override:** generateImages uses `path.join(process.cwd(), 'public', 'images', 'insights')` internally. To use press-release paths, construct the ImagePipelineInput with `imagesBaseUrl` pointing to the press-releases URL and ensure the local paths are under `public/images/press-releases/`. The generateImages function writes to a hardcoded path — you may need to rename/move the files after generation OR accept the insights/ path for v1 and note as TODO(v2) to add outputDir param to generateImages. Use TODO(v2) path for v1 — do not modify generateImages.
